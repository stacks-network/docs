## Documentation Structure

The documentation is organized into five main sections (Spaces), each designed to guide you through different aspects of Stacks:

### **Learn**
Deep technical explanations and conceptual breakdowns covering the core components of Stacks. Understand the "why" and "how" behind Stacks' architecture, consensus mechanisms, and relationship with Bitcoin.

### **Build**
Practical resources for developers including quickstarts, how-to guides, integration examples, Clarinet tutorials, Stacks.js documentation, and code samples. Everything you need to start building today.

### **Operate**
Complete setup and configuration guides for running Stacks infrastructure including nodes, miners, and signers. Learn how to contribute to network security and decentralization.

### **Reference**
Technical specifications and API documentation including Clarity language details, protocol specifications, SDK references, and comprehensive tooling documentation.

### **Tutorials**
Step-by-step walkthroughs that guide you through building complete applications from start to finish, helping you master Stacks development through hands-on practice.

## How the Docs Work

The Stacks documentation is built using **GitBook** with two-way synchronization to our [GitHub repository](https://github.com/stacks-network/docs). This means changes made in either location are automatically reflected in the other.

### Structure

Each documentation Space consists of:

- **`.gitbook/`** ‚Äî Assets folder containing images, PDFs, and other media
- **`README.md`** ‚Äî The landing page for that section
- **`SUMMARY.md`** ‚Äî Table of contents defining the section's structure
- **Content files** ‚Äî Markdown files containing the actual documentation

All documentation content is written in standard Markdown format, making it easy to read, edit, and contribute.

## Contributing

We welcome and encourage contributions from the community! The Stacks documentation is only as strong as the collective knowledge and effort of its contributors.

### What We're Looking For

Contributions of all kinds are valuable:

- **Fix typos and grammatical errors** ‚Äî Even small improvements matter
- **Clarify confusing explanations** ‚Äî Help make complex topics accessible
- **Add missing guides or tutorials** ‚Äî Share your knowledge with others
- **Update outdated information** ‚Äî Keep the docs current and accurate
- **Improve code examples** ‚Äî Better examples lead to better understanding
- **Add visual aids** ‚Äî Diagrams, screenshots, and illustrations enhance learning
- **Suggest structural improvements** ‚Äî Help us organize information more effectively

### How to Contribute

You can contribute in two ways:

#### Option 1: Direct Pull Request

1. **Fork** the [docs repository](https://github.com/stacks-network/docs)
2. **Create a branch** for your changes
3. **Make your edits** to the relevant Markdown files
4. **Test** your changes locally if possible
5. **Submit a pull request** with a clear description of your changes

This method is ideal if you're comfortable with Git and want to make direct contributions.

#### Option 2: Create an Issue

1. **Open an issue** in the GitHub repository
2. **Describe** the improvement, correction, or addition you're suggesting
3. **Provide context** about why this change would be valuable
4. A documentation maintainer will review and implement if appropriate

This method works well for suggesting improvements without making the changes yourself.

### Contribution Guidelines

When contributing, please:

- **Be clear and concise** ‚Äî Write in plain language when possible
- **Use proper Markdown formatting** ‚Äî Maintain consistency with existing docs
- **Include code examples** ‚Äî Show, don't just tell
- **Test your changes** ‚Äî Ensure links work and formatting renders correctly
- **Follow the existing structure** ‚Äî Keep content organized within the appropriate Space
- **Respect the tone** ‚Äî Maintain a helpful, professional, and welcoming voice
