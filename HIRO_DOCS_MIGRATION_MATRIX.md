# Hiro Docs Migration Matrix

This document is the gating artifact for the Hiro-to-Stacks docs migration.
No section should merge until the `recommendation` and `review` fields have
been confirmed by Alex, Rafa, and Hugo.

Guiding rules:

- Use first-class Stacks docs placement, not a temporary `/hiro` subtree.
- Keep English as the repo source of truth.
- Publish Spanish through GitBook Translations after English approval.
- Use GitBook built-in OpenAPI blocks for API references.
- Keep existing Hiro-hosted runtime endpoints on `hiro.so`.

Review checklist for each `keep` section:

- Accuracy and relevance confirmed
- Duplicate coverage on Stacks docs checked
- Legacy/deprecated wording removed
- Internal links updated to final Stacks paths
- GitBook preview checked
- GitBook admin dependency captured if an OpenAPI spec must be registered
